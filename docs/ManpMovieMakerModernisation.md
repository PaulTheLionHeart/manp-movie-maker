# ManpMovieMaker Modernisation

## Part 1: Specification

## 1. Purpose

ManpMovieMaker is a companion utility for ManpWIN that turns animation frame sequences into polished video files.

Its primary purpose is to help users create high-quality fractal animations from PNG frames generated by ManpWIN, with optional interpolation, soundtrack support, FFmpeg integration, and future presentation features such as titles or magnification overlays.

---

## 2. Core Goals

### 2.1 Create High-Quality Movies

ManpMovieMaker should produce high-quality MP4 output using FFmpeg, avoiding the older MPEG2 limitations built into ManpWIN.

### 2.2 Add Soundtracks

The program should allow users to add audio tracks, such as MP3 files, to generated movies.

#### Future Enhancements

* Fade audio in
* Fade audio out
* Trim audio to movie length
* Avoid abrupt audio cut-off at the end of animations

### 2.3 Movie Timing and Ending

The program should assist users in producing professionally finished movies.

Features include:

* Calculate movie duration from frame count and frame rate
* Optional final-frame hold period
* Final-frame hold specified in seconds rather than frame counts
* Default hold period: 2 seconds
* Final-frame hold should work regardless of animation type
* Audio fade-out should naturally complement the final-frame hold period

### 2.4 Interpolate Frames

The program should optionally create intermediate frames between ManpWIN-generated PNG frames.

This is especially useful for:

* Slow zoom animations
* Expensive deep zoom renders
* Derivative slope animations
* Art Matrix Cubic and other slow fractals

Practical limits:

* 1 to 4 intermediate frames usually gives good results
* More than 4 may reduce clarity, especially at 1080p or higher

### 2.5 Hide Directory Management

The user should not need to manually manage working folders.

The program should automatically create temporary and intermediate directories as needed and clean them up afterwards where appropriate.

### 2.6 Reduce Manual FFmpeg Work

The user should not need to manually edit or run FFmpeg scripts for common movie creation tasks.

Preferred future model:

* Store FFmpeg location in a configuration file
* Call FFmpeg internally
* Optionally still generate scripts for advanced users

---

## 3. Current Pain Points

* Directory management feels clunky
* FFmpeg workflow requires too much manual handling
* Audio can be cut abruptly when movie length is shorter than the soundtrack
* Limited validation of input folders
* User must understand internal processing steps
* Documentation is minimal

---

## 4. Desired User Workflow

Ideal simple workflow:

1. Select folder containing ManpWIN PNG frames
2. Select output movie filename
3. Optionally select soundtrack
4. Choose interpolation count
5. Choose audio fade options
6. Press Create Movie

The program should then:

* Scan input frames
* Validate basic consistency
* Calculate total frames
* Calculate movie duration
* Generate intermediate frames if requested
* Call FFmpeg
* Produce final movie
* Report success or errors clearly

### User Interface Principles

* Keep dialogs compact and suitable for laptop screens
* Use simple edit boxes with sensible defaults
* Express timing values in seconds rather than frame counts
* Hide temporary directory management from the user
* Minimise the number of decisions required to create a movie
* Preserve access to advanced options without cluttering the primary workflow

---

## 5. Input Validation

The program should check:

* Input folder exists
* PNG files exist
* Files are sequential or usable
* Image dimensions are consistent
* More than one animation sequence is not accidentally mixed in one folder
* Output folder is writable
* FFmpeg is available if direct movie creation is requested

Validation should be friendly, not obstructive.

---

## 6. Animation Types

ManpWIN can produce different animation types. ManpMovieMaker should not assume every animation is a zoom.

Known types include:

* Zoom animations
* Julia parameter animations
* Formula/parameter animations
* Palette animations
* Rotation animations
* Mixed animations

Interpolation is safest for zoom animations and may be inappropriate for some parameter-based animations.

Interpolation is optional.

Users should be able to:

* Create intermediate frames and build a movie
* Build a movie directly from existing frames without interpolation

Movie creation and audio processing should work independently of the interpolation subsystem.

---

## 7. Future Presentation Features

Possible future additions:

* Opening title
* Formula name display
* Magnification overlay
* Credits
* Simple subtitle/text overlay
* Optional metadata extracted from ManpWIN PNG text chunks or script files

Formula name probably only needs to appear near the beginning.

Magnification could update periodically, such as once per second, rather than every frame.

---

## 8. Technical Modernisation Goals

Bring the codebase toward modern C++ standards for safety, clarity, and education.

Candidate improvements:

* Replace fixed-size path buffers with `std::filesystem::path`
* Use `std::string` or `std::wstring` instead of raw character arrays where practical
* Use `std::vector` for dynamic collections
* Use small classes or structs for movie settings
* Separate UI, image processing, and FFmpeg command generation
* Improve error handling
* Preserve existing working behaviour while modernising incrementally

---

## 9. Documentation Goals

Create clear user documentation covering:

* What ManpMovieMaker does
* Basic workflow
* How to prepare frames in ManpWIN
* How interpolation works
* When not to use interpolation
* How to add sound
* How audio fade works
* How FFmpeg is configured
* Troubleshooting common problems

---

## 10. Development Philosophy

Modernise incrementally.

Prefer:

* Understand first
* Plan carefully
* Make small safe changes
* Test each step
* Preserve working behaviour
* Improve usability without breaking existing workflows

Avoid large rewrites unless clearly justified.

# Part 2: Practical Approach to Updating ManpMovieMaker

This document describes the vision, requirements, development philosophy, and modernisation strategy for ManpMovieMaker.

The objective is to improve usability, simplify workflow, modernise the codebase, and preserve the program's proven functionality while avoiding unnecessary rewrites.

Development will proceed incrementally using a plan-first approach.

* Define development roadmap
